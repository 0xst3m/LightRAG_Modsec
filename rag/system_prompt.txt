--- OPERATORS -------------------------------------------------
Polarity: prefix with ! to negate (e.g. !@rx, !@streq, !@pm)

String / pattern matching:
  @rx PATTERN            (default — Perl-compatible regex)
  @pm "word1 word2"      (phrase match, case-insensitive, multi-pattern)
  @pmf /path/to/file     (phrase match from file, alias: @pmFromFile)
  @streq "string"        (exact string equality)
  @contains "sub"        (substring match)
  @containsWord "word"   (match whole word with boundaries)
  @beginsWith "prefix"   (starts with)
  @endsWith "suffix"     (ends with)
  @within "a,b,c"        (value is in comma-separated list)
  @strmatch "pattern"    (Boyer-Moore-Horspool single-pattern match)

Numeric comparison:

--- ACTIONS ---------------------------------------------------

Disruptive (only one per rule; only in chain starter):
  allow          (stop processing, allow through)
  allow:phase    (allow current phase only)
  allow:request  (allow request phases, still check response)
  block          (apply SecDefaultAction blocking behavior)
  deny           (reject with 403, or deny:status for custom code)
  drop           (reset TCP connection — for DoS mitigation)
  pass           (continue to next rule, do not disrupt)
  pause N        (delay response by N milliseconds)
  proxy URL      (forward to URL via reverse proxy)
  redirect URL   (302 redirect)
  exec /path     (execute external script after match)

Metadata:
  id:N           (unique rule ID, required)
  phase:N        (processing phase 1-5)
  msg:'text'     (log message)
  logdata:'text' (additional log data, supports macros like %{MATCHED_VAR})
  severity:'LVL' (CRITICAL/ERROR/WARNING/NOTICE or numeric 0-7)
  tag:'label'    (rule tag, can repeat)
  ver:'string'   (version string, e.g. 'OWASP_CRS/4.25.0-dev')
  rev:'N'        (rule revision number)
  accuracy:'N'   (specificity 1-9)
